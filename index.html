HTML Boiler plate

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">

  <!-- Your content goes here -->

    < !DOCTYPE html>

  • Declares the document type and version of HTML (HTML5 here).
  • Helps the browser render the page correctly.

    • <html lang="en">

  • The root element of the HTML document.
  • lang="en" specifies the language of the document (English).

    • <head>

  • Contains meta-information about the document (not visible on the page).
    • <meta charset="UTF-8">
    • Sets the character encoding for the page to UTF-8 (supports most characters).
    • <meta name="viewport" content="width=device-width, initial-scale=1.0">
    • Ensures responsive design on mobile devices by controlling layout scaling.
    • <title>
    • Sets the title of the page that appears in the browser tab.
    • Optional elements inside <
    • <link>=> To include CSS files.
    • <script>=>To include JavaScript files
    • <meta>=>Additional metadata (like description, keywords, author).

    • <body>

  • Contains all the visible content of the page, like text, images, buttons, etc.

Block Elements and Inline Element

    Block Elements

  • Always start on a new line (take up the full width available).
  • They stack vertically (one after another).
  • Can contain inline elements or other block elements.
  • example:<div>, <p>, <h1> to <h6>, <ul>, <li>, <section>, <header>, <footer>, <article>, <table>

    • Inline Elements

  • Do NOT start on a new line.
  • Only take up as much width as necessary.
  • Usually used inside block elements (cannot wrap block-level elements inside inline).
  • Example:<span>, <a>, <strong>, <em>, <b>, <i>, <img>, <label>

HTML Entities

HTML entities are used to display reserved characters or special symbols in HTML.

  • Why we need HTML entities:
  • Some characters like < (less than) and > (greater than) are reserved in HTML, so we cannot use them directly in our code.
  • HTML entities allow us to display special characters such as © (©), ® (®), & (&), " ("), and ' (').
  • They prevent HTML parsing errors when using reserved symbols inside content.
  • They improve readability and ensure proper display across different browsers.

Common HTML Entities Examples(add prefix==>&)

  • lt; → < (less than)
  • gt; → > (greater than)
  • amp; → & (ampersand)
  • quot; → " (double quote)
  • apos; → ' (single quote)
  • copy; → © (copyright symbol)
  • reg; → ® (registered trademark)
