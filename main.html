What is Internet?

  • The Internet is a global network of interconnected computers and devices that communicate with each other using standard protocols (mainly TCP/IP) to share data, information, and services.

How to Identifying any Computer on the Internet?

  • Every device connected to the Internet has a unique IP address.
  • The IP address acts like a postal address for the computer.
  • Communication between two computers always requires source IP → destination IP.
Example:
  1. You type example.com in your browser.
  2. Browser asks DNS → gives IP 93.184.216.34.
  3. Your computer sends a TCP packet (with destination IP = 93.184.216.34, port 80/443).
  4. Routers forward the packet → server receives it.
  5. Server replies back to your public IP.
  6. Connection established → now data flows.

Type of IP Address

  1. IPv4 (32-bit):=>IPv4 has both private and public ranges.
  2. IPv6 (128-bit):=>IPv6 also has both private (fc00::/7, fe80::/10) and public (2000::/3) ranges.
  3. Public IP Address:=>A Public IP address is your global, unique address on the internet.
  4. Private IP Address:=>A Private IP address is a local address used only within your own private network (e.g., your home Wi-Fi, your office network).

  • IPv4 is written as four octets (e.g., 192.168.0.1).
  • Each octet = 8 bits → values from 0–255.

IPV6

  • IPv6 stands for Internet Protocol version 6.
  • It is the successor of IPv4 and provides a much larger address space.
  • IPv6 addresses are 128 bits long, compared to 32 bits in IPv4.
  • Addresses are represented in hexadecimal and separated by colons (:).
  • IPv6 supports automatic address configuration (stateless auto-configuration).
  • It improves security with built-in IPsec support.
  • IPv6 eliminates the need for Network Address Translation (NAT).
  • It provides better routing efficiency and hierarchical addressing.
  • IPv6 supports multicast instead of broadcast, reducing network traffic.

IPv6 Address Types

Address Type Description Example
Unicast Identifies a single interface. Packets are delivered to one specific device. 2001:0db8::1
Multicast Identifies a group of interfaces. Packets are delivered to all members of the group. FF02::1
Anycast Assigned to multiple interfaces, but packets are delivered to the nearest one. 2001:0db8::2
Link-local Used for communication on the same link (local network segment). FE80::1
Global unicast Routable on the internet, similar to public IPv4 addresses. 2001:0db8:85a3::8a2e:0370:7334

Public IP Address

  • Public IP: Assigned by your Internet Service Provider (ISP) and is unique across the entire internet.
  • Private IP: Used within a private network and can be reused in different networks.
  • Public IP allows devices to communicate over the internet.
  • Private IP allows devices to communicate within a local network (LAN).
  • Private IP addresses are not routable on the internet.
  • Public IP can be dynamic (changes) or static (fixed).

Mac Address

  • MAC Address is a unique identifier assigned to a network interface card (NIC).
  • It is used for communication within a local network (LAN).
  • MAC addresses are 48 bits long (6 bytes) and usually written in hexadecimal format.
  • Format examples: 00:1A:2B:3C:4D:5E or 00-1A-2B-3C-4D-5E.
  • It is fixed by the manufacturer and generally cannot be changed (though some devices allow spoofing).
  • MAC addresses are used in protocols like Ethernet and Wi-Fi.
  • MAC is different from IP addresses — IP addresses can change, MAC addresses are hardware-specific.

MAC Address Structure

Part Length Description Example
OUI (Organizationally Unique Identifier) 24 bits (first 3 bytes) Identifies the manufacturer of the NIC 00:1A:2B
NIC Specific 24 bits (last 3 bytes) Identifies the specific device 3C:4D:5E

Types of MAC Addresses

  • Unicast MAC: Identifies a single network interface.
  • Multicast MAC: Identifies a group of interfaces.
  • Broadcast MAC: Sent to all devices on the local network (FF:FF:FF:FF:FF:FF).

Port Number

  • A port number is a 16-bit number used to identify a specific process or service on a device.
  • Port numbers allow multiple services to run on a single IP address.
  • Range of port numbers: 0 to 65535.
  • Used by protocols like TCP and UDP to establish communication.
  • Ports can be associated with well-known services (HTTP, FTP, etc.) or custom applications.

Types of Port Numbers

Type Port Range Description Examples
Well-Known Ports 0 - 1023 Reserved for standard services and protocols. HTTP (80), HTTPS (443), FTP (21), SMTP (25)
Registered Ports 1024 - 49151 Assigned for user applications or services not requiring special privileges. MySQL (3306), RDP (3389), Minecraft Server (25565)
Dynamic / Private Ports 49152 - 65535 Used for temporary or private connections, often assigned dynamically by the OS. Ephemeral ports for client-server connections

Additional Notes

  • TCP and UDP both use port numbers, but the same port can exist on both protocols separately.
  • Ports below 1024 often require administrative privileges to bind.
  • Port scanning is used to discover open ports on a device, commonly for security auditing.

DNS (Domain Name System)

DNS is the global, distributed system that translates human-friendly domain names (like www.google.com) into computer-friendly IP addresses (like 142.250.72.206). Without DNS, you'd have to memorize IP addresses to access websites.

Step 1: Checking the Local Cache

Your computer first checks if it already has the answer:

  • Browser Cache: Stores recent DNS lookups for speed.
  • Operating System (OS) Cache: Keeps a record of recently resolved domain names.
  • Router Cache: Some routers store DNS records temporarily to reduce network traffic.

If the IP address is found in any cache, the process stops and the browser connects immediately.

Step 2: The Recursive Resolver

  • Your computer queries a Recursive DNS Resolver, usually provided by your ISP or public resolvers like Google (8.8.8.8) or Cloudflare (1.1.1.1).
  • Recursive: The resolver does all the work to find the final IP address, traversing the DNS hierarchy for you.

Step 3: The Hierarchical Search

  1. Root Servers: The resolver starts with 13 logical root servers asking where the TLD servers are (e.g., .com, .org).
  2. TLD Servers: These servers know which authoritative servers manage specific domains (e.g., google.com).
  3. Authoritative Name Server: The final server provides the exact IP address for the requested domain.

Step 4: Returning the IP Address

  1. The authoritative server sends the IP back to the recursive resolver.
  2. The resolver caches the IP for a defined TTL (Time-To-Live).
  3. The resolver returns the IP to your OS, which then passes it to your browser.

Step 5: Establishing Connection

The browser can now connect directly to the IP address on the appropriate port (e.g., port 443 for HTTPS) to request the website.

Additional Knowledge About DNS

Types of DNS

  • Forward DNS: Resolves domain names to IP addresses.
  • Reverse DNS: Resolves IP addresses back to domain names.
  • Dynamic DNS: Updates DNS records automatically when IP addresses change.

Common DNS Record Types

Record Type Description Example
A Maps a domain to an IPv4 address. example.com → 93.184.216.34
AAAA Maps a domain to an IPv6 address. example.com → 2606:2800:220:1:248:1893:25c8:1946
CNAME Alias of one domain pointing to another domain. www.example.com → example.com
MX Mail exchange server for handling emails. example.com → mail.example.com
NS Specifies authoritative name servers for the domain. example.com → ns1.example.com
PTR Used for reverse DNS lookups. 93.184.216.34 → example.com
TXT Text records often used for SPF, DKIM, or verification. v=spf1 include:_spf.example.com ~all

DNS Caching and Security

  • Caching: Improves speed by storing recently looked-up addresses.
  • TTL (Time-To-Live): Defines how long a DNS record is cached.
  • DNS Spoofing/Poisoning: Malicious activity attempting to redirect DNS queries to fake IPs.
  • DNSSEC: A security protocol that ensures DNS responses are authentic.

Summary

DNS acts like the internet's phonebook. It allows users to access websites using human-readable names instead of memorizing IP addresses. The system is hierarchical, distributed, and optimized with caching and security mechanisms to ensure fast and reliable browsing.
